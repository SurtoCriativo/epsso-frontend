## 🎯 Visão Geral

A **EPSSO** é uma empresa brasileira líder em saúde e segurança ocupacional, atuando desde 2005 com excelência e inovação. Nossa plataforma digital foi desenvolvida para:

- 📊 **Apresentar** a história, serviços e valores da empresa
- 🎓 **Divulgar** nosso catálogo completo de cursos e treinamentos
- 📝 **Publicar** conteúdo educativo através do blog integrado
- 🤝 **Conectar** clientes e parceiros com nossas soluções

### 🛠️ Stack Tecnológica

A aplicação é uma **Single Page Application (SPA)** moderna, construída com:

```
React 18 + TypeScript + Vite + Tailwind CSS
```

## ⚙️ Configuração do Ambiente

### 📝 Variáveis de Ambiente

Crie um arquivo `.env` na raiz do projeto baseado em `.env.example`:

```env
# API WordPress
VITE_GET_POSTS_ENDPOINT=https://api.exemplo.com/posts
VITE_GET_CATEGORIES_ENDPOINT=https://api.exemplo.com/categories

# Serviço de E-mail
VITE_EMAIL_API_URL=https://api.exemplo.com/send-email

# Adicione outras variáveis conforme necessário
```

## 🚀 Como Começar

### 📦 Pré-requisitos
- Node.js 18+
- pnpm 8+

### 🔧 Instalação

```bash
# Clone o repositório
git clone https://github.com/epsso/frontend.git

# Entre no diretório
cd epsso-frontend

# Instale as dependências
pnpm install

# Configure as variáveis de ambiente
cp .env.example .env
```

### 💻 Comandos Disponíveis

| Comando | Descrição |
|---------|-----------|
| `pnpm dev` | Inicia servidor de desenvolvimento em http://localhost:5173 |
| `pnpm build` | Gera build otimizada para produção |
| `pnpm preview` | Visualiza build de produção localmente |
| `pnpm lint` | Executa linter no código |

## 🤝 Como Contribuir

### 📋 Guidelines de Contribuição

1. **🔀 Novas Rotas**
   - Registre em `src/routes/AppRoutes.tsx`
   - Use lazy loading para otimização

2. **🎨 Estilização**
   - Utilize Tailwind CSS consistentemente
   - Extraia componentes reutilizáveis

3. **✅ Validação**
   - Use Zod para schemas

4. **🌍 Internacionalização**
   - Mantenha textos em `src/constants/`
   - Facilite futuras traduções

5. **📝 Documentação**
   - Atualize `.env.example`
   - Documente novas funcionalidades

### 🔄 Fluxo de Trabalho

```bash
# 1. Crie uma branch
git checkout -b feature/nova-funcionalidade

# 2. Faça suas alterações
# 3. Execute o build
pnpm build

# 3. Adicione os arquivos que serão commitados
git add .

# 5. Commit com mensagem descritiva
git commit -m "feat: adiciona nova funcionalidade"

# 6. Push e abra PR
git push origin feature/nova-funcionalidade
```
